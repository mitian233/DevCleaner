# DevCleaner Project Roadmap

## Project Overview

DevCleaner is a command-line tool for cleaning cache files generated by development tools. It features a TUI interactive interface with automatic detection, intelligent cleaning, and statistics reporting.

## Development Phases

### Phase 1: Core Features (Completed)

**Goal:** Implement Minimum Viable Product (MVP)

#### 1.1 Project Infrastructure
- [x] Initialize TypeScript project
- [x] Configure development environment (tsconfig.json)
- [x] Install core dependencies:
  - `@clack/prompts` - TUI interaction library
  - `picocolors` - Terminal colors
  - `execa` - Process execution
- [x] Create project directory structure
- [x] Define type system (`types.ts`)

#### 1.2 Utility Functions Implementation
- [x] Directory size calculation (`utils/size.ts`)
  - Support for Windows PowerShell and Unix du commands
  - Optimized scanning for large directories (limit depth and file count)
  - Formatted output (B/KB/MB/GB)
- [x] Command execution wrapper (`utils/execute.ts`)
  - Command existence detection
  - Async execution wrapper
- [x] File system operations (`utils/format.ts`)
  - Path existence check
  - Directory deletion
  - Directory creation

#### 1.3 Cleaner Infrastructure
- [x] Base cleaner class (`tools/base.ts`)
  - Unified cleaning interface
  - Command-first strategy
  - Fallback deletion strategy
- [x] Cleaning strategy implementation:
  1. Prioritize using tool's built-in cleaning commands
  2. Safely delete directory when no command available
  3. Error handling and logging

#### 1.4 Core Package Manager Support
- [x] npm cleaner (`tools/npm.ts`)
  - Path: `%LOCALAPPDATA%\npm-cache` (Windows)
  - Command: `npm cache clean --force`
  - Risk: Low
- [x] Yarn cleaner (`tools/yarn.ts`)
  - Path: `%LOCALAPPDATA%\Yarn\Cache` (Windows)
  - Command: `yarn cache clean`
  - Risk: Low
- [x] pnpm cleaner (`tools/pnpm.ts`)
  - Path: `%LOCALAPPDATA%\pnpm\store` (Windows)
  - Command: `pnpm store prune`
  - Risk: Low

#### 1.5 Tool Detection and Registration
- [x] Tool detection logic (`tools/index.ts`)
  - Cache directory existence check
  - Cleaning command availability check
  - Directory size calculation
  - Filter valid tools
- [x] Tool registration mechanism
  - Unified management of all cleaners
  - Dynamic cleaner loading

#### 1.6 TUI Interaction Flow
- [x] Main program entry (`index.ts`)
  - Welcome interface
  - Detection progress display
  - Multi-select interaction
  - Risk confirmation
  - Cleaning execution
  - Result report

**Milestone:** Functional CLI tool supporting npm/yarn/pnpm cleaning

### Phase 2: Complete Features (Completed)

**Goal:** Support all mainstream development tools

#### 2.1 Programming Language Support
- [x] pip cleaner (`tools/pip.ts`)
  - Path: `%LOCALAPPDATA%\pip\Cache` (Windows)
  - Command: `pip cache purge`
  - Risk: Low
- [x] Gradle cleaner (`tools/gradle.ts`)
  - Path: `~/.gradle/caches`
  - Method: Manual deletion
  - Risk: Medium
- [x] Maven cleaner (`tools/maven.ts`)
  - Path: `~/.m2/repository`
  - Method: Manual deletion
  - Risk: High (requires double confirmation)
- [x] Cargo cleaner (`tools/cargo.ts`)
  - Path: `~/.cargo/registry`
  - Command: `cargo cache -a`
  - Fallback: Manual deletion
  - Risk: Medium
- [x] NuGet cleaner (`tools/nuget.ts`)
  - Path: `~/.nuget/packages`
  - Command: `dotnet nuget locals all --clear`
  - Risk: Medium
- [x] Go cleaner (`tools/go.ts`)
  - Path: `~/go/build`
  - Command: `go clean -cache`
  - Risk: Low

#### 2.2 System Cache Support
- [x] System temporary file cleaner (`tools/system.ts`)
  - Path: `%LOCALAPPDATA%\Temp` (Windows) or `/tmp` (Unix)
  - Method: Manual deletion
  - Risk: Low

#### 2.3 Risk Level System
- [x] Low risk (green label)
  - Automatically regenerated cache
  - No impact on project execution
- [x] Medium risk (yellow label)
  - Requires project rebuild
  - May need to re-download some dependencies
- [x] High risk (red label)
  - Requires re-downloading all dependencies
  - Extra confirmation workflow

#### 2.4 Cleaning Statistics
- [x] Before/after size comparison
- [x] Detailed cleaning report
- [x] Failed item error messages
- [x] Total cleaned space statistics

**Milestone:** Support for 10+ development tools, complete risk control and statistics

### Phase 3: Enhanced Features (Planned)

**Goal:** Improve user experience and functionality

#### 3.1 Command Line Argument Support
- [ ] Implement command line argument parsing
  - `--help` / `-h` - Display help information
  - `--version` / `-v` - Display version number
  - `--dry-run` - Preview mode (display only, no deletion)
  - `--all` - Automatically clean all detected caches
  - `--tool <name>` - Clean specified tool
  - `--json` - JSON format output
  - `--quiet` / `-q` - Quiet mode

Examples:
```bash
devcleaner npm yarn        # Clean only npm and yarn
devcleaner --dry-run       # Preview mode
devcleaner --all --yes     # Auto clean all, no confirmation needed
devcleaner --json          # JSON output
```

#### 3.2 Configuration File Support
- [ ] Support `.devcleanerrc` configuration file
  - JSON/YAML format
  - Project-level config (`.devcleanerrc`)
  - User-level config (`~/.devcleanerrc`)

Configuration example:
```json
{
  "tools": {
    "npm": { "enabled": true },
    "yarn": { "enabled": false },
    "maven": { "enabled": false }
  },
  "exclude": [
    "*.log",
    "*.tmp"
  ],
  "dryRun": false,
  "autoConfirm": false
}
```

#### 3.3 Cleaning Log Recording
- [ ] Record cleaning history
  - Timestamp
  - List of cleaned tools
  - Before/after sizes
  - Cleaning result (success/failure)
  - Error messages
- [ ] Log file location:
  - Windows: `%APPDATA%\devcleaner\logs\`
  - Unix: `~/.config/devcleaner/logs/`
- [ ] Log format:
  ```
  [2024-03-10 10:30:45] INFO  Starting cleanup
  [2024-03-10 10:30:46] INFO  npm Cache: Cleaned successfully, freed 1.38 MB
  [2024-03-10 10:30:47] ERROR Yarn Cache: Cleaning failed, error: Insufficient permissions
  [2024-03-10 10:30:48] INFO  Cleanup complete, total freed 253.11 MB
  ```

#### 3.4 History Query
- [ ] `devcleaner history` command
  - Display recent N cleanup records
  - Filter by time/tool/size
  - Statistics display
- [ ] `devcleaner stats` command
  - Total cleanup count
  - Total space freed
  - Most frequently cleaned tools
  - Cleanup success rate

#### 3.5 IDE Cache Support (Optional)
- [ ] VS Code cache cleaning
  - Path: Project-level `.vscode/`
  - Risk: Low
- [ ] JetBrains IDE cache cleaning
  - Path: Project-level `.idea/`
  - Risk: Low
- [ ] Other IDE config cleaning
  - `.vscode-test/`
  - `.history/`
  - `.DS_Store`

**Note:** IDE configuration cleaning should be project-level, not global

#### 3.6 Advanced Features
- [ ] Scheduled cleanup tasks
  - Cron expression configuration
  - Background execution
  - System tray notifications
- [ ] Enhanced preview mode
  - Display list of files to be deleted
  - File size distribution chart
  - Selective deletion
- [ ] Multi-language support
  - Chinese
  - English
  - Japanese
- [ ] Plugin system
  - Support custom cleaners
  - Third-party plugin extensions

**Milestone:** Provide enterprise-level cleaning tool with automation and customization support
